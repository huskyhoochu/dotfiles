## 축 ① 정보구조·결속

> **윤문에서 가장 중요한 축.** 표면 오류가 하나도 없어도 화제선이 흔들리면 글은 기계 번역체로 읽힌다. LLM·번역체 글이 가장 자주, 가장 크게 어긋나는 지점.

### 1-1. 주어 반복 [soft, 그러나 효과 큼]

직전 문장과 주어가 같고 혼동 위험이 없으면, 한국어는 주어를 **생략**한다. 영어식으로 매 문장 주어를 반복하면 어색하다.

- ✗ "이 기능은 데이터를 모은다. **이 기능은** 데이터를 정렬한다. **이 기능은** 결과를 저장한다."
- ✓ "이 기능은 데이터를 모아 정렬한 뒤 Ø 저장한다." / "이 기능은 데이터를 모은다. Ø 정렬하고 Ø 저장한다."

**생략 전 3중 점검** (자매 이론 — 초점화자 3층위): ① 인식·시점 주체가 그대로인가, ② 직전 문장의 중심(화제)과 같은 대상인가, ③ 같은 인칭·성격의 경쟁 후보가 문장에 없는가. **하나라도 아니면 생략하지 말고 명시**하라. 생략으로 지시 대상이 모호해지면 윤문이 아니라 개악이다.

### 1-2. 은/는 vs 이/가 [soft → 일부 hard]

- **은/는** = 화제·구정보·대조. **이/가** = 주어·신정보·초점.
- 처음 도입하는 대상 → 이/가. 이미 언급된 대상 → 은/는.
  - ✓ "어떤 연구가 발표됐다. 그 **연구는** 새 방법을 제안했다." (도입: 이/가 → 재언급: 은/는)
- **[hard]** 한 문장에 '은/는'이 둘 이상인데 대조 구조가 아니면 어색하다. 하나를 이/가로 바꾸거나 재구성.
- 재언급 대상에 이/가를 다시 붙이면(영어식 "the X" 반복) 화제선이 끊긴다 → 은/는으로.

### 1-3. 화제선 이탈 [soft]

문단은 하나의 중심(누구·무엇에 관한 글)을 유지해야 한다. 중심이 문장마다 튀면 글이 산만해진다.
- 점검: 각 문단에서 "이 단락은 X에 관한 것"의 X가 일관되는가? 불필요하게 새 화제로 갈아타지 않는가?
- "이것/이러한 점에서/그것은" 같은 모호한 지시어가 화제선을 흐리면, 가리키는 대상을 명시하거나 문장을 합쳐라.

## 축 ② 번역투

> 영어·일본어 구조가 한국어에 그대로 옮겨진 흔적. 한국어는 **행위자·능동·동사 중심**을 선호한다.

### 2-1. 이중피동 [hard — 자신 있게 고쳐라]

'되다/지다'가 겹친 형태. 거의 항상 틀린 것으로 본다.
- ✗ 검토되어지다 → ✓ 검토되다
- ✗ 해석되어진다 → ✓ 해석된다
- ✗ 불리워지다 → ✓ 불린다
- ✗ 수정되어지기도 한다 → ✓ 수정되기도 한다

### 2-2. 무생물 주어 + 피동 [soft — 장르 주의]

사람과 무생물이 함께 나오면, 한국어는 **사람·행위자를 주어로 세워 능동**으로 쓴다.
- ✗ "노견은 협회**에 의해** 보호되었다" → ✓ "협회가 그 노견을 보호했다"
- ✗ "이 결과는 연구진**에 의해** 분석되어지고 있다" → ✓ "연구진이 이 결과를 분석하고 있다"

**예외 (고치지 마라)**: 학술체의 "이 논문은 ~을 제시한다", 물리 현상의 "다리가 무너졌다", "시간이 부족하다" 등 무생물이 실제 주체이거나 학술 관습인 경우는 자연스럽다. 장르를 보고 판단하라.

### 2-3. '의'·'~에 대한'·'~에 있어서' 남용 [soft]

영어 명사구(of, about, in)를 그대로 옮긴 흔적. 명사구를 절·동사로 풀면 자연스럽다.
- ✗ "사용자**의** 데이터**의** 보안**의** 중요성" → ✓ "사용자 데이터를 안전하게 지키는 일이 왜 중요한지"
- ✗ "환경 보호**에 대한** 연구" → ✓ "환경을 어떻게 보호할지 다룬 연구" (또는 문맥상 "환경 보호 연구")
- ✗ "이 문제**에 있어서**" → ✓ "이 문제에서" / "이 문제는"
- '의'가 3번 이상 연쇄하면 거의 항상 풀어 쓸 수 있다.

### 2-4. 복수 '들' 남용 [soft]

한국어는 문맥으로 복수를 알 수 있으면 '들'을 생략한다. 영어 -s를 기계적으로 옮기지 마라.
- ✗ "여러 정보**들**을 수집한다" → ✓ "여러 정보를 수집한다"
- ✗ "사용자**들**의 요구사항**들**" → ✓ "사용자 요구사항"
- 예외: 개별성·분포를 강조할 때는 살린다 ("학생들이 저마다").

### 2-5. 기타 번역투 상투어 [hard]

- ✗ "~하는 데**에 있어서**" → ✓ "~하는 데"
- ✗ "~이 요구**되어진다**" → ✓ "~이 필요하다"
- ✗ "아무리 강조해도 지나치지 않다" (영어 cannot be overemphasized) → 문맥에 맞게 자연스럽게
- ✗ "가지다" 남용 ("중요성을 가진다") → ✓ "중요하다"

## 축 ③ 어미·리듬·문체

> LLM은 한 번 고른 종결어미를 끝까지 반복하고 문장을 계속 이어붙인다(읽는 부담을 못 느끼므로). 후처리로 비교적 쉽게 보정된다.

### 3-1. 레지스터(문체) 일관성 [hard]

한 글 안에서 종결 체계를 섞지 마라.
- ✗ "~합니다. 그런데 이건 좀 다르**다**." (합니다체 ↔ 한다체 혼용)
- ✗ "~해요. 하지만 너는 ~**한다**." (존대 ↔ 비존대)
- 글 전체의 지배적 체계를 파악해 거기에 맞춘다. 어느 쪽으로 통일할지는 글쓴이가 더 많이 쓴 쪽을 따른다.

### 3-2. 종결어미 변주 [soft]

같은 어미('~다', '~이다', '~것이다')가 3~4문장 이상 연속되면 단조롭다. 일부를 의문·감탄·길이 변화로 바꿔 리듬을 준다.
- 예외: 보도문·기술 매뉴얼은 의도적 단조로움이 적합하니 강제하지 마라.

### 3-3. 만연체 분절 [soft]

한 문장이 50자를 크게 넘거나 두 줄을 넘기면 분할을 검토한다. 종속절이 겹겹이 쌓인 문장은 끊어 읽으면 명료해진다.
- 분할 지점은 **정보 단위·화제 전환 경계**에 맞춘다(③과 ①의 연계).
- 예외: 학술 초록, 법률 문서는 장문이 관습이니 일괄 분할 금지.

## 축 ④ 어휘·조사 정밀성

> LLM이 상대적으로 잘 처리하는 축(국소 패턴). 그래도 최종 점검 가치는 있다.

### 4-1. 격 조사 호응 [hard]

주어-서술어, 목적어-서술어, 부사어-서술어의 짝이 맞아야 한다.
- ✗ "나는 책**이** 읽었다" → ✓ "나는 책**을** 읽었다"
- ✗ "결코 그렇다" → ✓ "결코 그렇지 않다" (부사-서술어 호응)
- ✗ "여간 어렵다" → ✓ "여간 어렵지 않다"

### 4-2. 겹말 [hard, 단 관습 예외]

같은 뜻이 불필요하게 중복된 말.
- ✗ 사전에 미리 → ✓ 미리 (또는 사전에)
- ✗ 완전히 전부 다 → ✓ 전부
- ✗ 역전 앞, 과반수 이상, 속 내막, 박수를 치다(→손뼉을 치다/박수하다)
- **예외 (고치지 마라)**: 사전 등재·관습화된 겹말은 허용. "장인어른"(존대), "처갓집", "외갓집". 외국 지명의 "도시마섬"처럼 한국어에서 의미 보충이 필요한 경우도 둔다.

### 4-3. 사동/피동 정밀성 [soft]

"누가 실제 행위자인가, 책임을 어디에 두는가"로 결정한다. 글쓴이 의도를 모르면 강제로 바꾸지 마라.
- "A가 B를 C하게 하다"(사동) vs "B가 C되다"(피동)는 강조점이 다르다.

### 4-4. 미세 조사 선택 [soft]

여러 선택지가 문법적으로 허용되고 어감만 다른 경우가 많다. 기본값만 두고 톤에 맞춰 조정.
- 장소+존재 → '에' ("평창에 있다"), 장소+동작 → '에서' ("평창에서 만나다")
- 으로서(자격) vs 으로써(수단)

## 축 ⑤ 사용자 확립 규칙 (전부 hard)

> 사용자가 자기 문서 윤문 과정에서 연쇄 교정으로 직접 확립한 규칙(2026-07). **전부 hard이며,
> "글쓴이의 목소리" 보존 원칙보다 우선한다** — 글쓴이 본인이 정한 규칙이기 때문이다. 원문이
> 이렇게 쓰여 있어도 고친다.

### 5-1. 한영 혼용 금지 — 사고 흐름은 한국어로만

생각의 전개에 한국어와 영어를 경계 없이 섞지 않는다. "단어 자체로서의 영어 용어"와 "표현
차원의 혼용"을 명확히 구분한다.

- **용어 표기 두 가지만 허용**: 번역+gloss("탈출 밸브(escape hatch)"), 또는 backtick
  (`self-mod`, `exit 2` — 코드·함수·변수·식별자).
- **하이브리드 조어 금지**: ✗ "핫-무장", "도구-shaped였다", "미매치", "순-양(net-positive)".
- **영어 동명사·동사의 한국어 용언화 금지**. 기술 용어는 backtick 명사로 언급하고 동사
  자리는 한국어 동사로 받는다:
  - ✗ 도그푸딩되었다 → ✓ `dogfooding`을 거쳤다
  - ✗ 머지되었을 때 → ✓ `merge`가 끝났을 때
  - ✗ 프로브했다 → ✓ 프로브를 보냈다 (명사 '프로브'는 허용)
  - ✗ 토크나이즈된 단어 → ✓ 토큰으로 분해한 단어
  - ✗ 매치되지 않았다 → ✓ 걸리지 않았다 (개념 명사 '매칭'은 허용)
- 정착한 외래 명사(프로토콜, 레코드, 티켓, 밸브)는 허용. 문제는 명사가 아니라 **용언화와
  조어**다.

### 5-2. 콜론·세미콜론·줄표(—)는 한국어 문장에 쓰지 않는다

영어 구두점의 의도를 한국어 문형으로 재작성한다. 기계적으로 지우는 게 아니라 그 구두점이
하던 일을 문장이 하게 만든다.

- **콜론(나열 예고·부연)** → 별도 문장으로: ✗ "가져옵니다: (표)" → ✓ "가져오면 다음과
  같습니다. (표)" / ✗ "산문입니다: 규칙 파일, …" → ✓ "산문입니다. 규칙 파일, … 같은
  것들입니다."
- **세미콜론(병렬 접속)** → 문장 분리 또는 연결어미: ✗ "산문이고; 스펙은 …" → ✓ "산문이고,
  스펙은 …" 또는 두 문장.
- **줄표 삽입구** → 쉼표·괄호·별도 문장: ✗ "학계는 — 서로 인용하지 않은 채 — 수렴했다" →
  ✓ "학계는 대체로 서로를 인용하지 않은 채 수렴했다."
- **줄표 라벨 구분**("제목 — 부제", "**항목** — 설명") → 마침표("가설 1. …"), 쉼표, 또는
  조사 연결("**core**는 …입니다").

### 5-3. 한자어 절제

순우리말로 충분히 바꿀 수 있는 한자어는 바꾼다. 기술 용어(판정·차단·강제·불변식 등 대체 시
의미가 흐려지는 것)는 유지한다.

- ✗ 출고하다 → ✓ 내놓다 · ✗ 과잉 차단 → ✓ 지나친 차단 · ✗ 과소집계 → ✓ 적게 세다
- ✗ 상시 차단 → ✓ 늘 막다 · ✗ 경화하다 → ✓ 굳히다

### 5-4. 피동형 명사 표제 금지

"확인됨", "도전받음", "시사만 함" 같은 피동·명사형 라벨은 부자연스럽다. 자연스러운
명사구나 문장으로 바꾼다.

- ✗ "**확인됨 — 기계가 잡으면서.**" → ✓ "**가설이 맞은 지점.** 기계가 사람이 놓친 것을
  잡았습니다."
- ✗ "**도전받음**" → ✓ "**가설이 흔들린 지점.**"

### 5-5. 쉼표 과다·시적 마무리 금지

쉼표를 겹쳐 리듬을 만드는 문장, 여운을 남기는 시적 종결은 쓰지 않는다. 담백한 평서로
끝낸다.

- ✗ "이것을 여기, 공개적으로, 미리 적어 둡니다 — 숨긴 공백이 아니라 의무로 읽히도록." →
  ✓ "이 다짐을 미리 공개해 둡니다. 숨긴 공백이 아니라 의무로 읽혀야 하기 때문입니다."

### 5-6. 자기 지칭형 메타 문장 금지

문서가 스스로의 성격을 선언하는 문장("이 포스트는 공학 로그입니다", "발표문이 아니라 공학
로그로 씁니다")은 삭제한다. 성격은 내용으로 보여 준다.

### 기계 검증 (수정 후 필수)

축 ⑤는 패턴이 명확해 grep으로 잔존을 잡을 수 있다. 수정 후 반드시 돌려라:

```bash
grep -nE " — |—|: |; " <file>          # 구두점 잔존 (코드 블록·frontmatter 제외하고 판단)
grep -nE "(되었|하는|했|된)" <file> | grep -iE "[a-z]{3,}(되|했|하)"   # 용언화 의심 지점
```

기계 스캔은 후보를 찾을 뿐이다. 각 후보가 진짜 위반인지(예: '나머지'는 '머지'의 오탐)는
문맥으로 판단한다.

## 핵심 원리: 왜 이렇게 고치는가

윤문 규칙을 외우지 말고 **원리**로 판단하라. 모든 규칙은 두 가지로 환원된다:

1. **한국어는 화제 중심 언어다.** 글의 중심(누구·무엇에 관한 글)을 정하고, 그 중심이 이어지는 한 주어를 생략하며(은/는으로 화제화), 중심이 바뀔 때만 새 주어를 이/가로 세운다. 축 ①이 여기서 나온다.

2. **한국어는 행위자·능동·동사 중심이다.** 사람을 주어로, 능동으로, 명사구보다 동사·절로 쓴다 — 영어의 무생물 주어·피동·명사구 선호가 한국어로 새어든 것을 되돌리는 일이다. 축 ②가 여기서 나온다.

축 ③·④는 이 두 원리로 정돈된 문장을 **표면에서 다듬는** 마무리다.

**판단이 설 때**: "사람이 이 내용을 한국어로 처음부터 썼다면 이렇게 썼을까?"를 물어라. 아니라면 고치고, 맞다면 두어라. 이것이 hard/soft를 가르는 실전 감각이다.
